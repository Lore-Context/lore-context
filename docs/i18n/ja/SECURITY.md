# セキュリティポリシー

Lore Context はメモリ、トレース、監査ログ、統合資格情報を取り扱います。セキュリティレポートは最優先事項として扱います。

## 脆弱性の報告

疑われる脆弱性、流出したシークレット、認証バイパス、データ露出、またはテナント分離の問題について公開イシューを開かないでください。

推奨される報告パス:

1. 利用可能な場合は、このリポジトリの **GitHub プライベート脆弱性報告**を使用してください。
2. プライベート報告が利用できない場合は、メンテナーにプライベートで連絡し、以下を含めてください:
   - 影響を受けるバージョンまたはコミット、
   - 再現手順、
   - 想定される影響、
   - 実際のシークレットまたは個人データが関与しているかどうか。

信頼性の高い報告には 72 時間以内に応答することを目指しています。

## サポート対象バージョン

Lore Context は現在 1.0 前のアルファソフトウェアです。セキュリティ修正は最初に `main` ブランチを対象とします。タグ付きリリースは、公開リリースがダウンストリームオペレーターによって積極的に使用されている場合、対象パッチを受け取ることがあります。

| バージョン | サポート |
|---|---|
| v0.4.x-alpha | ✅ アクティブ |
| v0.3.x 以前 | ❌ プレリリース内部のみ |

## 組み込みハードニング（v0.4.0-alpha）

アルファ版には以下の多層防御コントロールが含まれています。オペレーターはデプロイでこれらがアクティブであることを確認してください。

### 認証

- **API キーベアラートークン**（`Authorization: Bearer <key>` または `x-lore-api-key` ヘッダー）。
- **ロール分離**: `reader` / `writer` / `admin`。
- **プロジェクトスコープ**: `LORE_API_KEYS` JSON エントリには `projectIds: ["..."]` 許可リストを含めることができます。変更には一致する `project_id` が必要です。
- **空キーモードは本番環境でフェイルクローズ**: `NODE_ENV=production` でキーが設定されていない場合、API はすべてのリクエストを拒否します。
- **ループバックバイパスを削除**: 以前のバージョンは `Host: 127.0.0.1` を信頼していました。v0.4 はソケットレベルのリモートアドレスのみを使用します。

### レート制限

- **IP ごとおよびキーごとのデュアルバケットリミッター**（認証失敗バックオフ付き）。
- **デフォルト**: 未認証パスで IP ごとに 60 req/min、認証済みキーで 600 req/min。
- **60 秒内で 5 回の認証失敗 → 30 秒ロックアウト**（429 を返します）。
- 設定可能: `LORE_RATE_LIMIT_PER_IP`、`LORE_RATE_LIMIT_PER_KEY`、`LORE_RATE_LIMIT_DISABLED=1`（開発のみ）。

### ダッシュボード保護

- **HTTP Basic Auth ミドルウェア**（`apps/dashboard/middleware.ts`）。
- **本番起動は** `DASHBOARD_BASIC_AUTH_USER` と `DASHBOARD_BASIC_AUTH_PASS` なしでは開始を拒否します。
- `LORE_DASHBOARD_DISABLE_AUTH=1` は本番環境以外でのみ有効です。
- サーバーサイド admin キーフォールバックは**削除されました**: ダッシュボードプロキシが上流 API 資格情報を注入する前に、ユーザーは Basic Auth で認証されている必要があります。

### コンテナハードニング

- すべての Dockerfile が非 root の `node` ユーザーで実行されます。
- `apps/api/Dockerfile` と `apps/dashboard/Dockerfile` が `/health` に対して `HEALTHCHECK` を宣言します。
- `apps/mcp-server` は stdio のみ — ネットワークリスナーなし — で `HEALTHCHECK` を宣言しません。

### シークレット管理

- **ハードコードされた資格情報ゼロ。** すべての `docker-compose.yml`、`docs/deployment/compose.private-demo.yml`、`.env.example` のデフォルトは `${VAR:?must be set}` 形式を使用します — 明示的な値なしで起動が速やかに失敗します。
- `scripts/check-env.mjs` がプレースホルダー値（`read-local`、`write-local`、`admin-local`、`change-me`、`demo`、`test`、`dev`、`password`）を `NODE_ENV=production` 時に拒否します。
- すべてのデプロイドキュメントとサンプル README がリテラルデモ資格情報から洗浄されました。

### ガバナンス

- **すべてのメモリ書き込みでリスクタグスキャン**: API キー、AWS キー、JWT トークン、秘密鍵、パスワード、メールアドレス、電話番号が検出されます。
- **6 状態ステートマシン**と明示的な合法遷移テーブル。不正な遷移はスローします。
- **メモリポイズニングヒューリスティクス**: 同一ソース支配 + 命令動詞パターンマッチング → `suspicious` フラグ。
- **不変の監査ログ**がすべての状態遷移で追記されます。
- 高リスクコンテンツは `candidate` / `flagged` に自動ルーティングされ、レビューされるまでコンテキスト構成から保留されます。

### MCP ハードニング

- すべての MCP ツール入力は呼び出し前に **zod スキーマに対してバリデーション**されます。バリデーション失敗はサニタイズされた問題リストと共に JSON-RPC `-32602` を返します。
- **すべての変更ツール**は少なくとも 8 文字の `reason` 文字列を必要とし、スキーマで `destructiveHint: true` を表示します。
- 上流 API エラーは MCP クライアントに返される前に**サニタイズ**されます — 生の SQL、ファイルパス、スタックトレースが除去されます。

### ログ

- **構造化 JSON 出力**とハンドラーチェーン全体での `requestId` 相関。
- `content`、`query`、`memory`、`value`、`password`、`secret`、`token`、`key` に一致するフィールドの**自動リダクション**。メモリレコードとクエリの実際のコンテンツはログに書き込まれません。

### データ境界

- `agentmemory` アダプターは初期化時に上流バージョンをプローブし、非互換時に警告します。`LORE_AGENTMEMORY_REQUIRED=0` はアダプターを上流に到達できない場合のサイレント縮退モードに切り替えます。
- `apps/api` リクエストボディパーサーが `LORE_MAX_JSON_BYTES` 上限（デフォルト 1 MiB）を強制します。サイズ超過のリクエストは 413 を返します。
- Postgres 接続プールが `statement_timeout: 15000` を設定してクエリ時間を制限します。
- `LORE_REQUEST_TIMEOUT_MS`（デフォルト 30 秒）がすべてのリクエストハンドラーを制限します。タイムアウトは 504 を返します。

## デプロイガイダンス

- 設定された `LORE_API_KEYS` なしで Lore をリモートに公開しないでください。
- `reader` / `writer` / `admin` キーの**ロール分離**を優先してください。
- 本番環境では `DASHBOARD_BASIC_AUTH_USER` と `DASHBOARD_BASIC_AUTH_PASS` を**常に設定**してください。
- **`openssl rand -hex 32` でキーを生成してください**。サンプルに示されたプレースホルダー値を決して使用しないでください。
- 生の `agentmemory` エンドポイントをプライベートに保ち、Lore を通じてのみアクセスしてください。
- ループバック以外の公開に対しては、ダッシュボード、ガバナンス、インポート/エクスポート、同期、監査ルートをネットワークアクセスコントロールレイヤー（Cloudflare Access、AWS ALB、Tailscale ACL など）の背後に置いてください。
- **本番環境で API を起動する前に `node scripts/check-env.mjs` を実行してください。**
- 本番 `.env` ファイル、プロバイダー API キー、クラウド資格情報、顧客コンテンツを含む評価データ、プライベートメモリエクスポートを**決してコミットしないでください**。

## 開示タイムライン

確認された高影響の脆弱性について:

- 0 日: レポートを確認。
- 7 日: トリアージと重大度分類をレポーターと共有。
- 30 日: 調整済み公開開示（または相互合意による延長）。
- 30 日以上: 該当する場合、中程度以上の重大度に対して CVE 発行。

低重大度の問題については、次のマイナーリリース内での解決を見込んでいます。

## ハードニングロードマップ

フォローアップリリースで予定されている項目:

- **v0.5**: OpenAPI / Swagger 仕様。`pnpm audit --high`、CodeQL 静的解析、dependabot の CI 統合。
- **v0.6**: Sigstore 署名コンテナイメージ、SLSA プロバナンス、長期トークンの代わりに GitHub OIDC 経由の npm 公開。
- **Future hosted hardening**: KMS エンベロープ暗号化による `risk_tags` フラグ付きメモリコンテンツの保存時暗号化。
