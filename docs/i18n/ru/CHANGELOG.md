Все существенные изменения в Lore Context документируются здесь. Формат основан на

## [v0.4.0-alpha] — 2026-04-28

Первая публичная альфа. Завершает производственный спринт по усилению безопасности, который превратил MVP с неудавшимся аудитом в release-candidate альфу. Все пункты аудита P0 устранены, 12 из 13 пунктов P1 устранены (один частично — см. Примечания), 117+ тестов проходят, сборка всего монорепозитория чистая.

### Добавлено

- **`packages/eval/src/runner.ts`** — настоящий `EvalRunner` (`runEval` / `persistRun` /
  `loadRuns` / `diffRuns`). Eval теперь может запускать сквозную оценку извлечения на
  наборе данных пользователя и сохранять запуски как JSON для обнаружения регрессий во времени.
- **`packages/governance/src/state.ts`** — шестисостояний конечный автомат управления
  (`candidate / active / flagged / redacted / superseded / deleted`) с явной таблицей
  допустимых переходов. Недопустимые переходы вызывают исключение.
- **`packages/governance/src/audit.ts`** — вспомогательный метод добавления в неизменяемый
  аудит-журнал, интегрированный с типом `AuditLog` из `@lore/shared`.
- **`packages/governance/detectPoisoning`** — эвристика обнаружения отравления памяти
  с использованием доминирования одного источника (>80%) и сопоставления паттернов
  императивных глаголов.
- **`packages/agentmemory-adapter/validateUpstreamVersion`** — зондирование upstream-версии
  на основе semver с собственной реализацией сравнения (без новых зависимостей). Учитывает
  `LORE_AGENTMEMORY_REQUIRED=0` для тихого перехода в режим деградации.
- **`packages/mif`** — поля `supersedes: string[]` и `contradicts: string[]` добавлены
  в `LoreMemoryItem`. Цикл сохраняется в форматах JSON и Markdown.
- **`apps/api/src/logger.ts`** — структурированный JSON-логгер с автоматическим
  редактированием чувствительных полей (`content` / `query` / `memory` / `value` /
  `password` / `secret` / `token` / `key`). `requestId` передаётся через всю цепочку
  обработки запроса.
- **`apps/dashboard/middleware.ts`** — промежуточное ПО HTTP Basic Auth. Запуск в
  продакшне отклоняется без `DASHBOARD_BASIC_AUTH_USER` и `DASHBOARD_BASIC_AUTH_PASS`.
- **`scripts/check-env.mjs`** — валидатор env в режиме продакшн. Отказывается запускать
  приложение, если любое значение окружения соответствует шаблону заглушки (`read-local`,
  `write-local`, `admin-local`, `change-me`, `demo`, `test`, `dev`, `password`).
- **Ограничение частоты** — двойной бакет per-IP и per-key с backoff при ошибках
  аутентификации (5 ошибок за 60 с → блокировка 30 с → ответ 429). Настраивается через
  `LORE_RATE_LIMIT_PER_IP`, `LORE_RATE_LIMIT_PER_KEY`, `LORE_RATE_LIMIT_DISABLED`.
- **Graceful shutdown** — обработчики SIGTERM/SIGINT завершают текущие запросы до 10 с,
  сбрасывают ожидающие записи Postgres, закрывают пул, принудительно выходят через 15 с.
- **Индексы базы данных** — B-tree индексы по `(project_id)` / `(status)` /
  `(created_at)` для `memory_records`, `context_traces`, `audit_logs`, `event_log`,
  `eval_runs`. GIN индексы по jsonb `content` и `metadata`.
- **MCP zod валидация ввода** — каждый MCP-инструмент теперь запускает `safeParse`
  против zod-схемы конкретного инструмента; ошибки возвращают JSON-RPC `-32602` с
  санированным списком проблем.
- **MCP `destructiveHint` + обязательный `reason`** — каждый мутирующий инструмент
  (`memory_forget`, `memory_update`, `memory_supersede`, `memory_redact`) требует
  `reason` не менее 8 символов и указывает `destructiveHint: true`.
- 117+ новых тест-кейсов в `apps/api`, `apps/mcp-server`, `packages/eval`,
  `packages/governance`, `packages/mif`, `packages/agentmemory-adapter`.
- Многоязычная документация: README на 17 языках в `docs/i18n/<lang>/`.
- `CHANGELOG.md` (этот файл).
- `docs/getting-started.md` — быстрый старт для разработчиков за 5 минут.
- `docs/api-reference.md` — справочник по конечным точкам REST API.
- `docs/i18n/README.md` — руководство участника перевода.

### Изменено

- **`packages/mif`** версия конверта `"0.1"` → `"0.2"`. Импорт с обратной совместимостью.
- **`LORE_POSTGRES_AUTO_SCHEMA`** по умолчанию `true` → `false`. Продакшн-развёртывания
  должны явно включать автоприменение схемы или запускать `pnpm db:schema`.
- **`apps/api`** парсер тела запроса теперь потоковый с жёстким ограничением размера
  полезной нагрузки (`LORE_MAX_JSON_BYTES`, по умолчанию 1 МБ). Слишком большие запросы
  возвращают 413.
- **Аутентификация loopback** изменена: удалена зависимость от заголовка URL `Host`;
  обнаружение loopback теперь использует только `req.socket.remoteAddress`. В продакшне
  без настроенных API-ключей API завершается ошибкой и отклоняет запросы (ранее: тихо
  предоставлялись права администратора).
- **Скопированные API-ключи** теперь должны предоставлять `project_id` для
  `/v1/memory/list`, `/v1/eval/run` и `/v1/memory/import` (ранее: неопределённый
  `project_id` обходил проверку).
- **Все Dockerfile** теперь запускаются от непривилегированного пользователя `node`.
  `apps/api/Dockerfile` и `apps/dashboard/Dockerfile` объявляют `HEALTHCHECK`.
- **`docker-compose.yml`** `POSTGRES_PASSWORD` теперь использует `${POSTGRES_PASSWORD:?must
  be set}` — запуск завершается ошибкой без явного пароля.
- **`docs/deployment/compose.private-demo.yml`** — тот же паттерн обязательного значения
  или ошибки.
- **`.env.example`** — все демо-значения по умолчанию удалены и заменены заглушками
  `# REQUIRED`. Задокументированы новые переменные для ограничения частоты, таймаута
  запроса, ограничения полезной нагрузки, обязательного режима agentmemory, Basic Auth
  dashboard.

### Исправлено

- **Уязвимость обхода аутентификации loopback** (P0). Злоумышленник мог отправить
  `Host: 127.0.0.1` для подделки обнаружения loopback и получения роли администратора
  без API-ключа.
- **Confused-deputy в прокси dashboard** (P0). Прокси dashboard внедрял `LORE_API_KEY`
  для неаутентифицированных запросов, предоставляя права администратора любому, кто мог
  получить доступ к порту 3001.
- **Защита от перебора** (P0). Демо-ключи (`admin-local`, `read-local`, `write-local`),
  указанные в README/`.env.example`, могли перебираться неограниченно; ограничение
  частоты и удаление значений по умолчанию теперь защищают от этого.
- **Аварийный сбой при разборе некорректного `LORE_API_KEYS` JSON** — процесс теперь
  завершается с понятной ошибкой вместо вывода трассировки стека.
- **OOM при большом теле запроса** — тела выше настроенного лимита теперь возвращают
  413 вместо аварийного завершения Node-процесса.
- **Утечка ошибок MCP** — ошибки upstream API, содержащие сырой SQL, пути к файлам или
  трассировки стека, теперь санируются до `{code, generic-message}` перед доставкой
  MCP-клиентам.
- **Аварийный сбой разбора JSON в dashboard** — некорректные JSON-ответы больше не
  приводят к аварийному завершению UI; ошибки отображаются как видимое пользователю состояние.
- **MCP `memory_update` / `memory_supersede`** ранее не требовали `reason`; теперь
  это принудительно проверяется zod-схемой.
- **Пул Postgres**: `statement_timeout` теперь установлен на 15 с; ранее было
  неограниченное время выполнения запросов при некорректных jsonb-запросах.

### Безопасность

- Все выводы аудита P0 (обход loopback / аутентификация dashboard / ограничение частоты
  / демо-секреты) устранены. Public release notes summarize the hardening work.
- `pnpm audit --prod` сообщает об отсутствии известных уязвимостей на момент релиза.
- Демо-учётные данные удалены из всех шаблонов развёртывания и примеров README.
- Образы контейнеров теперь запускаются от непривилегированного пользователя по умолчанию.

### Примечания / Известные ограничения

- **Частичный P1-1**: `/v1/context/query` сохраняет разрешительное поведение scoped-ключей,
  чтобы не нарушать существующие тесты потребителей. Другие затронутые маршруты
  (`/v1/memory/list`, `/v1/eval/run`, `/v1/memory/import`) применяют `project_id`.
  Отслеживается для v0.5.
- **Размещённая мультитенантная облачная синхронизация** не реализована в v0.4.0-alpha.
  Только локальные и Compose-приватные развёртывания.
- **Качество перевода**: локализации README сгенерированы LLM и чётко помечены;
  PR от сообщества для улучшения каждой локали приветствуются (см.
  [`docs/i18n/README.md`](../README.md)).
- **Спецификация OpenAPI / Swagger** ещё не упакована. Поверхность REST задокументирована
  в прозе в [`docs/api-reference.md`](api-reference.md). Отслеживается для v0.5.

## [v0.0.0] — предрелиз

Внутренние вехи разработки, не выпущенные публично. Реализовано:

- Каркасы пакетов рабочего пространства (TypeScript монорепозиторий, pnpm workspaces).
- Общий конвейер сборки/тестирования TypeScript.
- Система типов память / контекст / eval / аудит в `@lore/shared`.
- Граница адаптера `agentmemory`.
- Локальный REST API с маршрутизатором и составителем контекста.
- Персистентность на основе JSON-файла + опциональное хранилище Postgres с инкрементальным
  upsert.
- Потоки детализации / редактирования / замены / удаления памяти с явным жёстким удалением.
- Реальный учёт использования памяти (`useCount`, `lastUsedAt`).
- Обратная связь по трассам (`useful` / `wrong` / `outdated` / `sensitive`).
- MIF-подобный импорт/экспорт JSON + Markdown с полями управления.
- Набор регулярных выражений для сканирования секретов.
- Прямые метрики eval на основе сессий; сравнительные eval-запуски провайдеров; листинг
  eval-запусков.
- Защита API-ключами с разделением ролей reader/writer/admin.
- Очередь проверки управления; API аудит-журнала.
- API-обслуживаемый HTML dashboard; автономный Next.js dashboard.
- Демо-данные; генерация конфигурации интеграции.
- Приватная упаковка Docker/Compose.
- Legacy + официальный SDK stdio MCP транспорт.
