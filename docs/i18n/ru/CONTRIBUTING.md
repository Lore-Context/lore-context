# Участие в разработке Lore Context

Благодарим за улучшение Lore Context. Этот проект — управляющий уровень контекста для AI-агентов
на стадии альфы, поэтому изменения должны сохранять приоритет локальной работы, возможность аудита
и безопасность развёртывания.

## Кодекс поведения

Этот проект следует [Contributor Covenant](../../CODE_OF_CONDUCT.md). Участвуя, вы соглашаетесь
соблюдать его.

## Настройка среды разработки

Требования:

- Node.js 22 или новее
- pnpm 10.30.1 (`corepack prepare pnpm@10.30.1 --activate`)
- (Опционально) Docker для пути Postgres
- (Опционально) `psql`, если вы предпочитаете применять схему самостоятельно

Основные команды:

```bash
pnpm install --frozen-lockfile
pnpm build
pnpm test
pnpm smoke:api
pnpm smoke:mcp
pnpm smoke:dashboard
pnpm smoke:postgres   # требует docker compose up -d postgres
pnpm run doctor
```

Для работы с отдельными пакетами:

```bash
pnpm --filter @lore/api test
pnpm --filter @lore-context/server test
pnpm --filter @lore/eval test
pnpm --filter @lore/governance test
pnpm --filter @lore/mif test
pnpm --filter @lore/agentmemory-adapter test
```

## Ожидания от Pull Request

- **Держите изменения сфокусированными и обратимыми.** Одна задача на PR; один PR на задачу.
- **Добавляйте тесты** для изменений поведения. Предпочтительны реальные утверждения,
  а не снимки состояния.
- **Запускайте `pnpm build` и `pnpm test`** перед запросом проверки. CI тоже их запускает,
  но локально быстрее.
- **Запускайте соответствующий дымовой тест** при изменении поведения API, dashboard, MCP,
  Postgres, import/export, eval или развёртывания.
- **Не коммитируйте** сгенерированные артефакты сборки, локальные хранилища, файлы `.env`,
  учётные данные или приватные данные клиентов. `.gitignore` покрывает большинство путей;
  если вы создаёте новые артефакты, убедитесь, что они исключены.
- **Оставайтесь в рамках вашего PR.** Не рефакторируйте несвязанный код попутно.

## Архитектурные ограничения

Эти ограничения не подлежат обсуждению для v0.4.x. Если PR нарушает одно из них, ожидайте
запроса на разделение или переработку:

- **Локальная работа остаётся приоритетной.** Новая функция должна работать без размещённого
  сервиса или стороннего SaaS.
- **Никаких новых обходов поверхности аутентификации.** Каждый маршрут остаётся защищён
  API-ключом + ролью. Loopback не является исключением в продакшне.
- **Никакого прямого раскрытия `agentmemory`.** Внешние вызывающие получают доступ к памяти
  только через конечные точки Lore.
- **Целостность аудит-журнала.** Каждая мутация, влияющая на состояние памяти, записывает
  аудит-запись.
- **Завершение с ошибкой при отсутствии конфигурации.** Запуск в режиме продакшн отклоняется,
  если обязательные переменные окружения являются заглушками или отсутствуют.

## Сообщения коммитов

Lore Context использует небольшой, самоуверенный формат коммитов, вдохновлённый рекомендациями
ядра Linux.

### Формат

```text
<тип>: <краткое описание в повелительном наклонении>

<необязательное тело, объясняющее, почему это изменение нужно и какие компромиссы применяются>

<необязательные трейлеры>
```

### Типы

- `feat` — новая возможность, видимая пользователю, или конечная точка API
- `fix` — исправление ошибки
- `refactor` — реструктуризация кода без изменения поведения
- `chore` — гигиена репозитория (зависимости, инструменты, перемещение файлов)
- `docs` — только документация
- `test` — только изменения тестов
- `perf` — улучшение производительности с измеримым воздействием
- `revert` — откат предыдущего коммита

### Стиль

- **Строчные буквы** для типа и первого слова описания.
- **Без точки в конце** строки описания.
- **≤72 символа** в строке описания; перенос тела на 80.
- **Повелительное наклонение**: "fix loopback bypass", а не "fixed" или "fixes".
- **Почему, а не что**: diff показывает, что изменилось; тело должно объяснять почему.
- **Не включайте** трейлеры `Co-Authored-By`, атрибуцию AI или строки signed-off-by,
  если это не требуется явно пользователем.

### Полезные трейлеры

При необходимости добавляйте трейлеры для фиксации ограничений и контекста проверяющего:

```text
Constraint: Public deployments must keep /health unauthenticated for uptime checks
Confidence: high
Scope-risk: narrow
Tested: pnpm test, pnpm smoke:api
Not-tested: Cloudflare Access integration
Closes: #123
```

### Пример

```text
fix: loopback bypass — remove host-header trust, fail closed in production

Previous versions trusted the Host header for loopback detection, allowing a
remote attacker to spoof "Host: 127.0.0.1" and obtain admin role with no API
key when LORE_API_KEYS was unset. Loopback detection now uses
req.socket.remoteAddress only, and production with empty keys returns 401.

Constraint: Local development without keys still works on loopback
Confidence: high
Scope-risk: narrow (auth path only)
Tested: pnpm test (added loopback-bypass test case), pnpm smoke:api
```

## Гранулярность коммитов

- Одно логическое изменение на коммит. Проверяющие могут откатывать атомарно без
  сопутствующего ущерба.
- Сжимайте тривиальные исправления (`typo`, `lint`, `prettier`) в родительский коммит
  перед открытием или обновлением PR.
- Рефакторинг нескольких файлов допустим в одном коммите, если у них общая причина.

## Процесс проверки

- Мейнтейнер проверит ваш PR в течение 7 дней при обычной активности.
- Устраняйте все блокирующие комментарии перед повторным запросом проверки.
- Для неблокирующих комментариев допустимо ответить в тексте с обоснованием или
  создать последующую задачу.
- Мейнтейнеры могут добавить метку `merge-queue` после одобрения PR; не делайте
  rebase или force-push после применения этой метки.
