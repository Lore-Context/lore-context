# Участь у розробці Lore Context

Дякуємо за покращення Lore Context. Цей проект є площиною управління контекстом
AI-агентів на стадії альфа, тому зміни повинні зберігати операцію з пріоритетом
локальності, аудитованість та безпеку розгортання.

## Кодекс поведінки

Цей проект дотримується [Contributor Covenant](CODE_OF_CONDUCT.md). Беручи участь,
ви погоджуєтеся дотримуватись його.

## Налаштування середовища розробки

Вимоги:

- Node.js 22 або новіше
- pnpm 10.30.1 (`corepack prepare pnpm@10.30.1 --activate`)
- (Опціонально) Docker для шляху Postgres
- (Опціонально) `psql`, якщо ви бажаєте застосувати схему самостійно

Основні команди:

```bash
pnpm install --frozen-lockfile
pnpm build
pnpm test
pnpm smoke:api
pnpm smoke:mcp
pnpm smoke:dashboard
pnpm smoke:postgres   # вимагає docker compose up -d postgres
pnpm run doctor
```

Для роботи з окремими пакунками:

```bash
pnpm --filter @lore/api test
pnpm --filter @lore/mcp-server test
pnpm --filter @lore/eval test
pnpm --filter @lore/governance test
pnpm --filter @lore/mif test
pnpm --filter @lore/agentmemory-adapter test
```

## Очікування від pull request

- **Тримайте зміни зосередженими та оборотними.** Одне питання на PR; один PR на
  питання.
- **Додавайте тести** для змін поведінки. Віддавайте перевагу реальним твердженням
  перед знімками.
- **Виконуйте `pnpm build` та `pnpm test`** перед запитом перевірки. CI також запускає
  їх, але локально швидше.
- **Виконуйте відповідний димовий тест** при зміні API, дашборду, MCP, Postgres,
  імпорту/експорту, eval або поведінки розгортання.
- **Не комітьте** згенерований вихід збірки, локальні сховища, файли `.env`,
  облікові дані або приватні клієнтські дані. `.gitignore` охоплює більшість шляхів;
  якщо ви створюєте нові артефакти, переконайтеся, що вони виключені.
- **Залишайтеся в межах свого PR.** Не рефакторуйте непов'язаний код мимохідь.

## Архітектурні обмеження

Вони є обов'язковими для v0.4.x. Якщо PR порушує одне, очікуйте запит на
розбиття або доопрацювання:

- **Локальність залишається основним принципом.** Нова функція повинна працювати без
  хостингового сервісу або стороніх SaaS залежностей.
- **Ніяких нових обходів поверхні автентифікації.** Кожен маршрут залишається за
  API ключем + роллю. Зворотний зв'язок не є особливим випадком у виробництві.
- **Ніякого прямого доступу до `agentmemory`.** Зовнішні виклики досягають пам'яті
  лише через ендпоінти Lore.
- **Цілісність журналу аудиту.** Кожна мутація, що впливає на стан пам'яті, записує
  запис аудиту.
- **Закриватися при відсутній конфігурації.** Запуск у виробничому режимі відмовляє,
  якщо обов'язкові змінні середовища є заповнювачами або відсутні.

## Повідомлення комітів

Lore Context використовує невеликий, суб'єктивний формат комітів, натхненний
рекомендаціями ядра Linux.

### Формат

```text
<тип>: <короткий опис у наказовому способі>

<необов'язкове тіло, що пояснює, чому ця зміна потрібна і які компроміси застосовуються>

<необов'язкові трейлери>
```

### Типи

- `feat` — нова видима для користувача можливість або API ендпоінт
- `fix` — виправлення помилки
- `refactor` — реструктуризація коду без зміни поведінки
- `chore` — гігієна репозиторію (залежності, інструментарій, переміщення файлів)
- `docs` — лише документація
- `test` — лише зміни тестів
- `perf` — покращення продуктивності з вимірюваним впливом
- `revert` — відкочування попереднього коміту

### Стиль

- **Нижній регістр** для типу та першого слова опису.
- **Без крапки** в кінці рядка опису.
- **≤72 символи** в рядку опису; перенос тіла на 80.
- **Наказовий спосіб**: "fix loopback bypass", не "fixed" або "fixes".
- **Чому важливіше ніж що**: diff показує що змінилося; тіло повинно пояснювати чому.
- **Не включайте** трейлери `Co-Authored-By`, атрибуцію AI або рядки signed-off-by
  без явного запиту користувача.

### Корисні трейлери

При необхідності додавайте трейлери для фіксації обмежень та контексту перевіряльника:

```text
Constraint: Public deployments must keep /health unauthenticated for uptime checks
Confidence: high
Scope-risk: narrow
Tested: pnpm test, pnpm smoke:api
Not-tested: Cloudflare Access integration
Closes: #123
```

### Приклад

```text
fix: loopback bypass — remove host-header trust, fail closed in production

Previous versions trusted the Host header for loopback detection, allowing a
remote attacker to spoof "Host: 127.0.0.1" and obtain admin role with no API
key when LORE_API_KEYS was unset. Loopback detection now uses
req.socket.remoteAddress only, and production with empty keys returns 401.

Constraint: Local development without keys still works on loopback
Confidence: high
Scope-risk: narrow (auth path only)
Tested: pnpm test (added loopback-bypass test case), pnpm smoke:api
```

## Гранулярність комітів

- Одна логічна зміна на коміт. Перевіряльники можуть атомарно відкочуватися без
  побічного збитку.
- Об'єднуйте тривіальні виправлення (`typo`, `lint`, `prettier`) в батьківський коміт
  перед відкриттям або оновленням PR.
- Багатофайлові рефакторинги допустимі в одному коміті, якщо вони мають єдину причину.

## Процес перевірки

- Супровідник перевірить ваш PR протягом 7 днів при типовій активності.
- Вирішуйте всі блокуючі коментарі перед повторним запитом перевірки.
- Для неблокуючих коментарів прийнятно відповісти в рядку з обґрунтуванням або
  наступним тікетом.
- Супровідники можуть додати мітку `merge-queue` після схвалення PR; не виконуйте
  rebase або force-push після застосування цієї мітки.
