# Журнал змін

Всі значущі зміни Lore Context задокументовано тут. Формат базується на
[Keep a Changelog 1.1.0](https://keepachangelog.com/en/1.1.0/), і цей проект
дотримується [Semantic Versioning 2.0.0](https://semver.org/spec/v2.0.0.html).

## [v0.4.0-alpha] — 2026-04-28

Перша публічна альфа. Завершує спринт виробничого зміцнення, який перетворив
MVP з невдалим аудитом на альфу-кандидат на випуск. Всі елементи аудиту P0 виправлено,
12 з 13 елементів P1 виправлено (один частково — див. Примітки), 117+ тестів
проходять, повна збірка монорепозиторію чиста.

### Додано

- **`packages/eval/src/runner.ts`** — реальний `EvalRunner` (`runEval` / `persistRun` /
  `loadRuns` / `diffRuns`). Eval тепер може виконувати наскрізну оцінку отримання на
  наборі даних користувача та зберігати запуски як JSON для виявлення регресій у часі.
- **`packages/governance/src/state.ts`** — шестистанний автомат станів управління
  (`candidate / active / flagged / redacted / superseded / deleted`) з явною таблицею
  допустимих переходів. Недопустимі переходи спричиняють виключення.
- **`packages/governance/src/audit.ts`** — допоміжна функція незмінного додавання до
  журналу аудиту, інтегрована з типом `AuditLog` з `@lore/shared`.
- **`packages/governance/detectPoisoning`** — евристика для виявлення отруєння пам'яті
  за допомогою домінування одного джерела (>80%) та зіставлення шаблонів наказових
  дієслів.
- **`packages/agentmemory-adapter/validateUpstreamVersion`** — перевірка upstream версії
  на основі semver з власною реалізацією порівняння (без нових залежностей). Враховує
  `LORE_AGENTMEMORY_REQUIRED=0` для тихого режиму деградації.
- **`packages/mif`** — поля `supersedes: string[]` та `contradicts: string[]` додані
  до `LoreMemoryItem`. Кругова поїздка збережена у форматах JSON та Markdown.
- **`apps/api/src/logger.ts`** — структурований JSON логер з автоматичною редакцією
  чутливих полів (`content` / `query` / `memory` / `value` / `password` / `secret` /
  `token` / `key`). `requestId` передається через весь ланцюг запиту.
- **`apps/dashboard/middleware.ts`** — проміжне ПЗ HTTP Basic Auth. Запуск у виробництві
  відмовляє без `DASHBOARD_BASIC_AUTH_USER` та `DASHBOARD_BASIC_AUTH_PASS`.
- **`scripts/check-env.mjs`** — валідатор середовища для виробничого режиму. Відмовляє
  в запуску застосунку, якщо будь-яке значення середовища відповідає шаблону
  заповнювача (`read-local`, `write-local`, `admin-local`, `change-me`, `demo`, `test`,
  `dev`, `password`).
- **Обмеження частоти** — подвійний кошик токенів per-IP та per-key з відступом при
  помилці автентифікації (5 невдалих спроб за 60с → блокування на 30с → відповідь 429).
  Налаштовується через `LORE_RATE_LIMIT_PER_IP`, `LORE_RATE_LIMIT_PER_KEY`,
  `LORE_RATE_LIMIT_DISABLED`.
- **Плавне завершення** — обробники SIGTERM/SIGINT завершують поточні запити протягом
  10с, скидають незавершені записи в Postgres, закривають пул, примусово виходять через
  15с.
- **Індекси бази даних** — B-tree індекси на `(project_id)` / `(status)` /
  `(created_at)` для `memory_records`, `context_traces`, `audit_logs`, `event_log`,
  `eval_runs`. GIN індекси на jsonb `content` та `metadata`.
- **MCP zod валідація вхідних даних** — кожен інструмент MCP тепер виконує `safeParse`
  по схемі zod для кожного інструменту; невдалі перевірки повертають JSON-RPC `-32602`
  з очищеними проблемами.
- **MCP `destructiveHint` + обов'язковий `reason`** — кожен мутаційний інструмент
  (`memory_forget`, `memory_update`, `memory_supersede`, `memory_redact`) вимагає
  рядок `reason` щонайменше 8 символів і відображає `destructiveHint: true`.
- 117+ нових тестових випадків у `apps/api`, `apps/mcp-server`, `packages/eval`,
  `packages/governance`, `packages/mif`, `packages/agentmemory-adapter`.
- Багатомовна документація: README на 17 мовах у `docs/i18n/<lang>/`.
- `CHANGELOG.md` (цей файл).
- `docs/getting-started.md` — 5-хвилинний швидкий старт для розробників.
- `docs/api-reference.md` — довідник REST API ендпоінтів.
- `docs/i18n/README.md` — посібник учасника перекладу.

### Змінено

- **`packages/mif`** версія конверту `"0.1"` → `"0.2"`. Сумісний імпорт.
- **`LORE_POSTGRES_AUTO_SCHEMA`** за замовчуванням `true` → `false`. Виробничі
  розгортання повинні явно увімкнути автоматичне застосування схеми або виконати
  `pnpm db:schema`.
- **`apps/api`** парсер тіла запиту тепер потоковий з жорстким обмеженням розміру
  корисного навантаження (`LORE_MAX_JSON_BYTES`, за замовчуванням 1 МіБ). Надмірно
  великі запити повертають 413.
- **Автентифікація зворотного зв'язку** змінена: видалено залежність від заголовка URL
  `Host`; виявлення зворотного зв'язку тепер використовує лише
  `req.socket.remoteAddress`. У виробництві без налаштованих API-ключів API блокується
  та відмовляє в запитах (раніше: тихо надавав адміністратора).
- **Ключі API з обмеженням до проекту** тепер повинні надавати `project_id` для
  `/v1/memory/list`, `/v1/eval/run` та `/v1/memory/import` (раніше: невизначений
  `project_id` обходив перевірку).
- **Всі Dockerfiles** тепер запускаються від непривілейованого користувача `node`.
  `apps/api/Dockerfile` та `apps/dashboard/Dockerfile` оголошують `HEALTHCHECK`.
- **`docker-compose.yml`** `POSTGRES_PASSWORD` тепер використовує
  `${POSTGRES_PASSWORD:?must be set}` — запуск без явного пароля завершується
  невдачею.
- **`docs/deployment/compose.private-demo.yml`** — той самий шаблон обов'язкового
  значення або відмови.
- **`.env.example`** — всі демонаборні значення за замовчуванням видалено та замінено
  заповнювачами `# REQUIRED`. Задокументовано нові змінні для обмеження частоти,
  тайм-ауту запиту, обмеження корисного навантаження, обов'язкового режиму agentmemory,
  базової автентифікації дашборду.

### Виправлено

- **Вразливість обходу автентифікації через зворотний зв'язок** (P0). Зловмисник міг
  надіслати `Host: 127.0.0.1`, щоб підробити виявлення зворотного зв'язку та отримати
  роль адміністратора без API-ключа.
- **Confused-deputy у проксі дашборду** (P0). Проксі дашборду вставляв `LORE_API_KEY`
  для неавтентифікованих запитів, надаючи адміністративні права будь-кому, хто міг
  досягти порту 3001.
- **Захист від перебору** (P0). Демо ключі (`admin-local`, `read-local`, `write-local`),
  показані в README/`.env.example`, могли бути перераховані нескінченно; обмеження
  частоти та видалення значень за замовчуванням тепер захищають від цього.
- **Аварійне завершення при невалідному `LORE_API_KEYS`** — процес тепер виходить з
  чітким повідомленням про помилку замість видачі трасування стека.
- **OOM через велике тіло запиту** — тіла понад налаштований ліміт тепер повертають
  413 замість аварійного завершення процесу Node.
- **Витік помилок MCP** — помилки upstream API, що містили необроблений SQL, шляхи до
  файлів або трасування стека, тепер очищуються до `{code, generic-message}` перед
  поверненням клієнтам MCP.
- **Аварійне завершення при парсингу JSON дашборду** — невалідні JSON-відповіді більше
  не аварійно завершують UI; помилки відображаються як видимий стан.
- **MCP `memory_update` / `memory_supersede`** раніше не вимагали `reason`; тепер це
  забезпечується схемою zod.
- **Пул Postgres**: `statement_timeout` тепер встановлено на 15с; раніше необмежений
  ризик часу запиту при невалідних jsonb запитах.

### Безпека

- Всі висновки аудиту P0 (обхід зворотного зв'язку / автентифікація дашборду /
  обмеження частоти / демо секрети) виправлено. Дивіться
  public release notes для повного журналу аудиту.
- `pnpm audit --prod` на момент випуску не повідомляє про відомі вразливості.
- Демо облікові дані видалено з усіх шаблонів розгортання та прикладів README.
- Зображення контейнерів тепер за замовчуванням запускаються без привілеїв root.

### Примітки / Відомі обмеження

- **Частковий P1-1**: `/v1/context/query` зберігає дозвільну поведінку ключа з
  обмеженням до проекту, щоб уникнути поломки існуючих споживчих тестів. Інші
  зачеплені маршрути (`/v1/memory/list`, `/v1/eval/run`, `/v1/memory/import`)
  забезпечують `project_id`. Відстежується для v0.5.
- **Хостована багатоорендна хмарна синхронізація** не реалізована в v0.4.0-alpha.
  Тільки локальні та Compose-приватні розгортання.
- **Якість перекладу**: локалізації README генеруються LLM та чітко позначені;
  PR від спільноти для вдосконалення кожної мови вітаються (дивіться
  [`docs/i18n/README.md`](../README.md)).
- **Специфікація OpenAPI / Swagger** ще не упакована. Поверхня REST задокументована
  прозою у [`docs/api-reference.md`](../../api-reference.md). Відстежується для v0.5.

Внутрішні етапи розробки, не були публічно випущені. Реалізовано:

- Спільний конвеєр збірки/тестування TypeScript.
