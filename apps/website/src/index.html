01 / Problem - the memory gap

Agents remember. Teams need proof.

Every coding agent, support bot, and research copilot is moving toward long-term memory. Most teams still cannot show the audit trail behind one answer.

01

Stale memory

Last week's API contract can still be recalled into this morning's pull request.

02

Unknown provenance

A quoted fact appears in an answer, but no one can explain which session produced it.

03

Unsafe writeback

Agents silently persist hallucinations, PII, and one-off conversations into shared memory.

04

Backend lock-in

Each vector store captures your data in a different schema, format, and retrieval path.

02 / System

A governed path between agents and memory.

Lore exposes a narrow safe surface: agents ask for context, Lore composes evidence, evals the result, logs the trace, and routes questionable memory through governance.

ingress

MCP clients

Claude Code, Cursor, Qwen Code, and local agents call the same stdio surface.

query

context.query

Typed retrieval returns ranked evidence with source and policy metadata.

compose

Composer

Memory is assembled into a budgeted context block with warnings and citations.

proof

Eval / trace / governance

Every answer path leaves measurable quality and review evidence.

store

Postgres + adapter

Persist locally with an audit log and bridge to agentmemory-compatible storage.

03 / Product surfaces

Built for operators, not memory hype.

F-01

Context Query

A single typed call your agent can rely on. Returns ranked memories with provenance, freshness, and policy state.

F-02

Memory Eval Playground

Replay queries against your own seed dataset. Tune retriever, reranker, and freshness cutoff.

F-03

Memory Observability

A live trace of every retrieval, every write, every redaction. Drill into any span.

F-04

Governance Review

Every writeback passes a policy gate. Humans can approve, redact, reject, or forget.

F-05

MIF-like Portability

Export the memory corpus, embeddings, provenance, and policy state without backend lock-in.

F-06

Private Deployment

Single Docker Compose. No telemetry, no phone-home, no proprietary embedding endpoint.

04 / Alpha - shipped this week

What is in v0.4 alpha.

Honest list. Built first, polished later. Everything below runs offline on your machine with a free Docker Compose.

REST API

/v1/context, /v1/memory, /v1/eval, typed and documented.

MCP stdio SDK transport

Drop into any MCP-compatible client without exposing raw backend tools.

Next Dashboard

Operator UI for memory health, traces, eval runs, and governance review.

Postgres incremental persistence

Audit logs, trace records, and memory rows survive local restarts.

Demo dataset

Seeded data makes the alpha inspectable without connecting private corpora.

Docker Compose

Private deploy path with Postgres and local services.
